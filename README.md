# Real-Time Transaction Monitoring Pipeline

## 🎯 Project Overview
This project simulates a real-time transaction monitoring system using Kafka, Python, and PostgreSQL.

The pipeline processes transaction data in real-time, applies validation, and stores clean data for analysis.

---

## ⚙️ Tech Stack
- Apache Kafka
- Python
- PostgreSQL
- Docker

---

## 🔄 Pipeline Flow
Producer → Kafka → Consumer → Validation → Database

---

## 📊 Data Source
The dataset is taken from Kaggle (Credit Card Fraud Detection) and transformed into streaming events.

---

## 📂 Project Structure
- producer: send transaction data to Kafka  
- consumer: process and validate data  
- database: database schema  
- images: project screenshots
